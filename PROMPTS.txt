/plan We need to upgrade the secured package to use the native cryptographic API's of the host OS. Ground rules, all tests must pass and cannot be modified, and envelope structures like HashedPassword and EncryptedData cannot be modified. The list of supported algorithms can be found in READEME.md The first task is we need to complete is to create bindings for MacOS CommonCrypto (or CryptoKit if possible in D) and Windows CNG and place them in the "bindings" folder, we already have OpenSSL bindings for Linux and FreeBSD, placed in the "bindings" folder. We should also add bindings for LibreSSL, GnuTLS, NSS, BoringSSL, and wolfSSL. Each binding needs a corresponding build configuration in dub.sdl. Since not all libraries will support all the specified algorithms we will need an additional build configuration named "polyfill" that leverages OpenSSL to fill-in what the other libraries do not. By default the library should use the OS provided API's and polyfill only when enabled. If an unsupported method is used and polyfill is not enabled a build error should be emitted where possible and a runtime error otherwise. Note that some OS's support algorithms like SHA3 on some versions and not others, this should be detected at runtime and a CryptographicException thrown if it is not available on this particular version of the OS. To make the code easier to reason about, the public API should only act as a algorithm availability detector and then dispatch to the correct private implementation method for the selected library configuration. This will primarily involve changes to the "_ex" methods as the non-"_ex" methods should always forward to the "_ex" methods. In all cases the specific private library implementation method must ONLY implement what is strictly necessary to interface to the library, all other code is to be left in the dispatch methods. Note that the implementation methods must be private as they are not intended to be used publicly.

2. The test suite should run all tests, but when an unsupported algo is detected it should print message noting that this algorithm is unsupported in this configuration. This is *only* modification allowed to unittests.
